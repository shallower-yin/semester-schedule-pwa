# 日程提醒

事项和习惯可以选择不提醒，或在开始时、提前 5/10/15/30/60 分钟、提前 1 天提醒。日期范围事项和每日习惯会按范围内每一天生成对应提醒。

纪念日、生日和节日也可以单独选择是否提醒，并设置提前 0 到 366 天、几点几分提醒。提前 0 天表示当天提醒。

## 两层提醒

1. 应用打开或恢复到前台时，每 30 秒检查本地 IndexedDB，并通过 Service Worker 显示系统通知。
2. 应用完全关闭时，Supabase `pg_cron` 每分钟调用 Edge Function，领取到期提醒并通过 Web Push 发送到已订阅设备；GitHub Actions 保留为手动排障入口。

后台调度受 Supabase 定时任务、浏览器推送服务、操作系统省电策略和网络影响，不承诺秒级准时。
单次 Web Push 发送失败时，调度器会在 15 分钟有效窗口内按分钟重试；失效的设备订阅会自动移除。
某个发生日期已经标记完成后，本地检查和云端派发都会跳过该次提醒；重复事项的其他日期不受影响。

## 检查当前设备

登录后打开右上角“账号与同步”，查看“系统提醒”：

- `当前设备已订阅`：应用关闭时也可以接收 Web Push。
- `尚未允许`：点击“启用提醒”，并在浏览器提示中选择允许。
- `已被浏览器阻止`：需要在浏览器的网站权限中重新允许通知。
- `仅应用打开时提醒`：当前设备尚未完成登录或云端订阅。
- `已允许，但未完成云端订阅`：点击“启用提醒”重新注册。

已经错过超过约 15 分钟的提醒不会补发。新建测试事项或习惯时，建议把开始时间设为当前时间之后 10 分钟，并选择提前 5 分钟提醒；测试纪念日时可选择今天、提前 0 天，并把提醒时间设为几分钟后。

Android Edge/Chrome 的后台 Web Push 依赖手机系统推送服务。启用过程会依次显示“浏览器权限 → 应用后台服务 → 手机系统推送服务 → 云端订阅”；任何阶段无响应都会在限定时间后退出，不会再永久卡住。若停在手机系统推送服务，请检查：

- Edge 或 Chrome 的通知权限是否允许；
- Edge 或 Chrome 是否被系统禁止后台运行或被省电策略限制；
- 当前 VPN/网络是否能访问 Google 推送服务；
- 手机系统是否具备可用的 Google Play 服务。

即使云端订阅失败，事项、习惯和纪念日仍可保存，并在应用保持打开时使用本地提醒；应用完全关闭后的提醒必须成功完成 Web Push 订阅。

## 用户要求

- 必须通过 HTTPS 在线地址使用。
- 每台手机或电脑都要登录同一账号。
- 首次打开事项提醒时，需要由用户点击允许通知。
- Android 和 Windows 均可使用 Edge 或 Chrome；Android 两者都可添加到手机桌面。
- 浏览器无痕模式、清除站点数据或退出登录会移除当前设备的推送订阅，需要重新允许。

## 安全

- VAPID 私钥和提醒调度令牌保存在 Supabase Edge Function Secrets，并由部署工作流同步。
- 后台任务由数据库定时器使用 Publishable key 调用 Edge Function，Edge Function 再使用调度令牌领取提醒。
- 浏览器只获得 VAPID 公钥和 Supabase Publishable key。
- 推送订阅表启用 RLS，用户只能管理自己的订阅。
- 调度 RPC 仅接受 SHA-256 哈希校验通过的随机令牌。
