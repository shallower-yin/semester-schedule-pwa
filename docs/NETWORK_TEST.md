# Supabase 接入前网络测试

正式配置 Supabase 前，应使用同一台 Android 手机和一台 Windows 电脑，在校园网、手机移动网络、家庭网络分别执行测试。

## 需要准备

- Supabase 项目 URL
- 项目的匿名公钥（anon key）
- 一个专用测试邮箱
- 最终部署站点的 HTTPS 地址

不要把 `service_role` 密钥放进网页、截图或测试记录。

## 每种网络需要验证

1. 浏览器能打开 Supabase 项目 API 地址。
2. 对 `/rest/v1/` 的 HTTPS 请求能收到响应，而不是 DNS、证书或超时错误。
3. 能提交注册邮件。
4. 测试邮箱能收到验证邮件。
5. 邮件验证链接能返回已部署网页。
6. 登录后可以读取一条仅属于测试账号的数据。
7. 关闭网络后页面仍能启动并读取 IndexedDB。
8. 离线新增事项，恢复网络后再次执行上传和拉取测试。
